# 1단계 (MVP) - 요구사항 설계

## 개요

초기 모델은 최소한의 기능으로 빠르게 출시하여 사용자 반응을 확인하는 것이 목표입니다. 복잡한 백엔드 없이 정적 웹사이트로 시작합니다.

## 핵심 요구사항

### 1.1 간단한 메뉴 추천
- 버튼 클릭 시 3초 이내 메뉴 표시
- 최소 30개 이상의 메뉴 데이터
- 카테고리 정보 표시 (한식, 일식, 중식, 양식, 분식, 기타)
- "다시 추천받기" 기능
- 반응형 디자인 (모바일/데스크톱)

### 1.2 깔끔한 UI/UX
- 중앙 배치된 큰 추천 버튼
- 큰 글씨로 결과 표시
- 3가지 이하 주요 색상
- 로딩 애니메이션
- 최소한의 텍스트

### 1.3 기본 카테고리 필터
- 카테고리 선택 옵션
- 선택한 카테고리 내에서만 추천
- 선택 안 하면 전체에서 추천
- 최소 6개 카테고리 지원

## 제외 사항 (1단계에서는 구현 안 함)

- 사용자 계정/로그인
- 백엔드 서버
- 데이터베이스
- 날씨 API 연동
- 위치 기반 추천
- 광고
- 가게 정보

## 성공 지표

- 페이지 로딩 시간 < 2초
- 추천 응답 시간 < 1초
- 모바일 접근성 100%
- 1-2주 내 배포 가능

## 사용자 시나리오

1. 사용자가 웹사이트 접속
2. (선택적) 카테고리 선택
3. "추천받기" 버튼 클릭
4. 메뉴 추천 결과 확인
5. 만족하지 않으면 "다시 추천받기"
6. 히스토리에서 이전 추천 확인 (LocalStorage)

## 데이터 요구사항

### 메뉴 데이터 구조
```javascript
{
  id: number,
  name: string,           // "김치찌개"
  category: string,       // "한식"
  tags: string[],         // ["따뜻한", "국물", "매운맛"]
  description: string     // "얼큰한 김치찌개"
}
```

### 필요한 메뉴 수
- 한식: 10개
- 일식: 5개
- 중식: 5개
- 양식: 5개
- 분식: 3개
- 기타: 2개
- **총 30개**

## 기술적 제약사항

- 외부 API 호출 없음
- 서버 없음 (정적 호스팅만)
- 데이터베이스 없음
- 사용자 데이터는 브라우저에만 저장
- 비용: $0

## 다음 단계로의 전환 조건

- 일일 방문자 100명 이상
- 사용자 피드백 수집 완료
- 추가 기능 요구사항 명확화
